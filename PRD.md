# PRD：PhotoQuote Map — 基于照片元数据与内容生成“国家级地图标注 + 金句”的单张海报（H5优先）

- 版本：V1.0（MVP 可收费）
- 形态：H5 网站（先做 Web，后续复用到微信小程序）
- 商业模式：首单免费；后续按次付费；每次付费包含“高清导出 + 3 次换一句”
- 核心承诺：不出现名人姓名；位置模糊化到“国家”；照片与元数据不做长期保存（仅临时处理并自动删除）

---

## 1. 背景与目标

### 1.1 背景
用户希望把旅行/纪念照片快速变成可分享的“故事感海报”：底部带拍摄日期、国家级地图标注、与画面相关的金句，并带二维码引导其他人“扫码生成同款”。

现有工具多为手动模板 + 手动文字，无法“自动读取拍摄信息 + 自动生成贴合内容的金句 + 自动合成地图标注”。

### 1.2 产品目标
1. 4 步闭环：**上传 →（预览）→ 付费 → 生成高清并保存**
2. 自动化：优先读取 EXIF（时间/坐标）；EXIF 缺失提供**手动填国家+日期**兜底
3. 内容生成：调用 AI API 基于“国家 + 日期 + 画面理解结果”生成金句（不署名）
4. 出图：输出单张海报（底部信息条 + 手绘风地图 + 国旗/国家名 + 金句 + 二维码）
5. 可变现：首单免费；后续按次；**单价需大于 2 次 API 调用成本并覆盖重生成**

### 1.3 非目标（MVP 不做）
- 城市/街道级定位与路线轨迹
- 视频处理
- 相册批量生成
- 名人名言真实引用与溯源（本产品不出现名人姓名）
- 账号体系与云相册（可后续迭代）

3. 活动合影：EXIF 不完整 → 手填国家+日期 → 生成统一风格纪念图

## 3. 核心体验与流程

### 3.1 体验原则
- **极简**：默认不需要登录；表单尽量少
- **先看后买**：允许低清预览（带水印）→ 付费解锁高清（降低纠纷）
- **失败可兜底**：EXIF 缺失可手动填；生成失败给“退回次数券/自动退款策略”
- **隐私透明**：明确“临时处理、到期删除、国家级模糊化”

### 3.2 主流程（推荐）
> 你原始设想是“上传→付费→生成→保存”。为降低退款与提升转化，MVP 推荐：  
> **上传→解析/补充→低清预览→付费→高清导出→保存**  
>（仍满足你的 4 步闭环目标，且更商业化。）

**流程：**
1. 上传照片（1 张）
2. 解析 EXIF：
   - 有：自动填日期；若有坐标→反查国家；无坐标→国家为空
   - 无：进入“手动补充国家+日期”
3. 生成低清预览（带水印 + 低分辨率）：
   - 显示海报结构与文案风格
4. 付费：
   - 首单免费：跳过支付直接出高清
   - 非首单：支付成功后出高清
5. 输出高清图：
   - 支持下载/保存
   - 提供“换一句（最多 3 次）”按钮（只重跑文案，不重跑视觉识别）
6. 分享：
   - 海报内嵌二维码：扫码进入上传页（带归因参数）

### 3.3 异常流程
- 图片过大/格式不支持：提示压缩或更换格式（支持 JPG/PNG/HEIC 转换策略见技术）
- EXIF 缺失：展示原因说明 + 手动填国家/日期 + 可继续生成
- AI 生成失败：提示重试；超过次数/持续失败则退回次数券
- 内容安全不通过：告知不可生成（并退回次数券/退款）

## 4. 功能需求（MVP）

### 4.1 上传与解析
**FR-01 上传照片**
- 支持：JPG / PNG（MVP 建议），可选支持 HEIC（iOS 原图常见）
- 单次仅 1 张
- 前端展示：文件名、尺寸、预览图

**FR-02 EXIF 读取**
- 读取字段：
  - 拍摄时间：DateTimeOriginal（若无则尝试其他日期字段）
  - GPS：Latitude/Longitude（若无则为空）
- 输出：
  - 日期（yyyy-mm-dd）
  - 坐标（可选）
- 若读取失败 → 触发手动补充

**FR-03 手动补充（兜底）**
- 国家：下拉搜索选择（必填）
- 日期：年月日（必填，精确到日）
- 说明文案：
  - “社交软件转发/保存可能会丢失拍摄信息。你可以上传原图，或手动补充国家与日期。”

### 4.2 国家反查与位置模糊化
**FR-04 GPS→国家**
- 当存在 GPS 时，服务端/前端反查国家（优先服务端以统一）
- 最终展示仅国家级：国家名（可选国旗）
- 不展示坐标、不展示城市

**FR-05 模糊化策略**
- 展示层面：仅国家名
- 内部处理：允许使用 GPS 仅用于反查国家，不落库

### 4.3 AI 内容生成
**FR-06 画面理解（Vision）**
- 输出结构化结果（示例）：
  - scene_tags：["beach","sunset","couple"]
  - mood：["warm","peaceful"]
  - objects：["sea","sky","people"]
  - short_caption：一句简短描述（可选）

**FR-07 金句生成（Text）**
- 输入：国家、日期、vision 结果（tags/mood/objects）
- 输出：金句 1 句（中文为主），长度控制：
  - 12–32 字（默认）
- 规则：
  - **不出现名人姓名**
  - 不包含敏感/攻击性内容
- 支持“换一句”：最多 3 次（仅重跑 FR-07，不重跑 FR-06）

### 4.4 海报渲染与导出
**FR-08 海报模板（手绘风）**
- MVP 至少 1 套模板：
  - 上：照片主图
  - 下：信息条（国家、日期、金句、二维码）
  - 左/右：手绘风小地图模块（国家位置点）
- 地图样式：手绘风（实现方式见技术）

**FR-09 低清预览**
- 分辨率：长边 1200px（示例）
- 叠加水印（产品名/“预览”字样）
- 不允许下载高清

**FR-10 高清导出**
- 分辨率：长边 3000px（示例，可配置）
- 输出格式：PNG 或 JPG（JPG 更省）
- 提供“下载”按钮

### 4.5 二维码回流与归因
**FR-11 二维码生成**
- 二维码内容：落地到 H5 上传页 URL，携带参数：
  - src=qr
  - tpl=template_id
  - mid=render_id（可选，用于归因/防刷）
- 海报右下角展示：
  - 二维码 + “扫码生成同款”

### 4.6 支付与计费
**FR-12 首单免费**
- 判定方式（MVP）：
  - 基于设备指纹/浏览器存储 + 风控（可后续提升）
  - 或基于微信环境 openid（若后续接微信授权）
- 首单免费仍限制“换一句”次数（建议同付费：3 次）

**FR-13 按次付费**
- 商品：一次“生成包”
  - 包含：高清导出 1 张 + 换一句 3 次
- 支付成功后解锁高清
- 单价原则：≥ 2 次 API 成本并覆盖重试与支付成本（见商业章节）

**FR-14 失败补偿**
- 生成失败（系统原因/模型错误）：退回次数券
- 内容违规（用户原因）：提示不可生成，退回次数券/不退（需在条款里写清）

## 5. 非功能需求（NFR）

### 5.1 性能
- 预览生成：目标 ≤ 10 秒（视 API 而定）
- 高清生成：目标 ≤ 15 秒
- 高峰并发：支持水平扩展（无状态服务）

### 5.2 稳定性
- AI API 超时与重试：最多 1–2 次指数退避
- 渲染失败可重试（不额外扣次数）

### 5.3 安全与隐私
- 明确：不做长期存储；仅为生成与下载**临时缓存**，到期删除（建议 10–30 分钟）
- HTTPS 全站
- 不记录照片原图到长期日志；日志仅保留 hash/错误码/耗时

### 5.4 合规
- 隐私政策：说明处理范围、临时保存时长、删除机制
- 内容安全：接入图片与文本安全检查（至少基础拦截）

## 8. 技术方案（MVP）

### 8.1 架构概览
- 前端（H5）：
  - 文件选择 + 预览
  - EXIF 读取（优先前端，减少无效上传）
  - 提交生成请求
  - 展示预览/结果 + 下载
- 后端（API 服务）：
  - 国家反查（GPS→国家）
  - Vision 分析
  - Quote 文案生成
  - Render 合成出图
  - 支付回调与次数管理
  - 临时文件存储与到期删除

### 8.2 数据流
1. Front: 读 EXIF → 得到 date/gps（可为空）
2. Front: 若缺失 → 用户手填 country/date
3. Front → Back: /analyze (上传图片或压缩图)
4. Back: vision → tags/mood/objects
5. Back: /quote → 生成金句
6. Back: /render → 合成预览/高清
7. Back: 返回临时下载链接（短时有效）

### 8.3 手绘风地图实现策略（MVP）
- 方案 A（推荐）：预制手绘世界底图（SVG/PNG）+ 国家点位数据（country→坐标）
  - 优点：稳定、风格统一、成本低
  - 说明：点位数据为“国家中心点/代表点”，不涉及精确定位
- 方案 B：调用地图服务截图后做风格化（不推荐 MVP，复杂且成本高）

### 8.4 临时存储与删除
- 将渲染结果存到对象存储/本地临时目录
- 设置 TTL：10–30 分钟自动删除
- 下载链接：签名 URL 或一次性 token

## 9. API 设计（建议契约）

### 9.1 POST /api/analyze
- 入参：image（file），meta（可选：date, gps, country）
- 出参：
  - render_id
  - vision：{scene_tags, mood, objects}
  - country（若可解析）
  - date（若可解析）

### 9.2 POST /api/quote
- 入参：render_id，country，date，vision
- 出参：
  - quote_text
  - quote_id

### 9.3 POST /api/render
- 入参：
  - render_id
  - quote_text
  - mode：preview | hd
  - template_id
- 出参：
  - image_url（短时有效）
  - expires_at

### 9.4 POST /api/pay/create
- 入参：render_id，product_id
- 出参：payment_url / payment_params

### 9.5 POST /api/pay/callback
- 支付回调：更新订单状态、扣减次数/解锁高清

### 9.6 POST /api/quote/regenerate
- 入参：render_id
- 出参：
  - new_quote_text
  - remaining_regen_count

## 11. 内容安全与风控

### 11.1 图片安全
- 色情/暴力/违法内容拦截
- 命中则：
  - 不生成

### 11.2 文案安全
- 过滤敏感词、歧视、攻击
- 输出长度与符号控制，避免“过长难看”

### 11.3 反刷
- 首单免费：设备指纹/频率限制
- 高频请求：限流与验证码（可后续）

## 12. 法务与文案（MVP 必须上）

### 12.1 隐私要点（需在 Policy 页面体现）
- 处理目的：生成海报
- 处理内容：照片（临时）、EXIF（时间/国家推断）、用户手填信息
- 存储：仅临时缓存用于生成与下载；到期自动删除（明确时长）
- 展示：仅国家级，不展示坐标/城市
- 用户权利：提示如何删除（自动删除）与联系渠道

### 12.2 免责声明要点
- 文案为 AI 生成，仅供分享与娱乐；不保证事实性
- 若用户上传不含 EXIF 或手填信息不准确，结果以用户输入为准
- 不提供名人引用与署名（避免误导）
